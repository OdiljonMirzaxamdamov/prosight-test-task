
# Prosight DEV Test Task

Backend API test task implemented with NestJS and TypeScript.

## Overview

This project provides:

- JWT authentication with 3 predefined users:
  - `admin`
  - `normal`
  - `limited`
- `GET /locus` endpoint
- filtering, sorting, and pagination
- optional sideloading of locus members
- role-based access control
- autogenerated OpenAPI documentation via Swagger
- basic automated tests

## Tech Stack

- NestJS
- TypeScript
- TypeORM
- PostgreSQL
- JWT / Passport
- Swagger / OpenAPI
- Jest / Supertest

## API Features

### Authentication

The API uses JWT authentication.

Predefined users:
- `admin` / `prosightAdmin111`
- `normal` / `prosightNormal222`
- `limited` / `prosightLimited333`

Login endpoint:
POST /auth/login

Example request body:
{
  "username": "admin",
  "password": "prosightAdmin111"
}

### Locus endpoint

GET /locus

Supported query parameters:

- `id` â€” integer

- `sortBy` â€” one of:
  - `id`

  - `locusStop`

Example requests:

GET /locus?regionId=86118093
GET /locus?sortBy=memberCount&sortOrder=DESC&limit=20
GET /locus?sideload=locusMembers&page=1&limit=10

## Access Rules

### Admin user

- can access all returned fields
- can use sideloading
- can access all data

### Normal user

- can access only data from `rl` table
- cannot use sideloading

### Limited user

- can access data only for the following `regionId` values:
  - 86118093
  - 86696489
  - 88186467
- cannot use sideloading

## Notes and Assumptions

- `assemblyId` was implemented according to the actual datasource schema type (`text` in the public database), even though the task text describes it as integer.
- Default `LIMITED_ALLOWED_REGION_IDS` matches the task statement. In the current public database these IDs return no rows, so the variable can be overridden locally if you want non-empty demo data for the limited user.
- API response fields use `camelCase`, while database columns use `snake_case`.
- For filtering by `regionId` and `membershipStatus`, ORM relations are used.
- No raw SQL was used in the implementation.
